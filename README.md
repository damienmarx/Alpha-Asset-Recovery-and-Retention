# FraudSim Lab - PPP Fraud Simulation & Detection Platform

> **Educational Security Research Tool** — Simulate and analyze PPP loan fraud scenarios with AI-powered scammer agents and real-time fraud detection.

## 🎯 Overview

**FraudSim Lab** is a comprehensive platform for security researchers, fraud analysts, and red teamers to:

- **Simulate** realistic PPP loan fraud scenarios using AI-powered scammer agents
- **Detect** fraud patterns through rule-based and LLM-powered analysis
- **Collaborate** with team members via comments and shared analysis
- **Customize** the dashboard with personalized themes and branding
- **Monitor** real-time call transcripts and alerts via WebSocket
- **Analyze** risk scores and fraud indicators for each call

## ✨ Key Features

### 🤖 AI-Powered Scammer Agent
- Uses **OpenAI GPT-3.5** or local **Ollama** LLM
- Realistic, conversational fraud narratives
- Customizable personas and tactics
- Natural language processing for dynamic responses

### 📞 VOIP Integration
- **Twilio** integration for real phone calls
- Inbound and outbound call support
- Real-time speech-to-text transcription
- Call recording and analysis

### 🔍 Multi-Layer Fraud Detection
- **Rule-based detection** — Pattern matching for known fraud indicators
- **LLM-based detection** — AI analysis of conversation for fraud risk
- **Real-time alerts** — Immediate notification of suspicious activity
- **Risk scoring** — Weighted analysis combining multiple detection methods

### 🎨 Customizable Dashboard
- **Prompt-based configuration** — Easy setup with interactive prompts
- **Color theming** — Customize primary, accent, background, and text colors
- **Branding** — Personalize dashboard title and logo
- **Real-time updates** — WebSocket-powered live transcript and alert streaming

### 👥 Collaboration Features
- **Team comments** — Annotate calls with analysis notes
- **User roles** — Admin, Analyst, Viewer access levels
- **Call history** — Browse and filter all calls
- **Detailed analytics** — Comprehensive call transcripts and alert logs

### 🌐 Tunnel Integration
- **Cloudflare tunnel** — One-command public URL exposure
- **Twilio webhook routing** — Automatic configuration
- **No port forwarding** — Secure tunnel without firewall changes

## 🚀 Quick Start

### Prerequisites

- **Python 3.9+**
- **Twilio account** (free trial with $15 credit)
- **OpenAI API key** (optional) or **Ollama** (local, free)
- **Git**

### 1. Clone Repository

```bash
git clone https://github.com/yourusername/ppp-fraud-lab.git
cd ppp-fraud-lab
```

### 2. Run Automated Setup

```bash
bash scripts/setup.sh
```

This will:
- Create a Python virtual environment
- Install all dependencies
- Prompt you for configuration (LLM, Twilio, dashboard theme)
- Generate `.env` file with your settings

### 3. Start the Application

```bash
python app.py
```

The dashboard will be available at: **http://localhost:8000**

**Default credentials:**
- Username: `admin`
- Password: `admin123`

### 4. (Optional) Expose via Tunnel

In a separate terminal:

```bash
bash scripts/setup_tunnel.sh
```

This will display a public URL like: `https://xxxx.trycloudflare.com`

## 📋 Configuration

### Environment Variables

All settings are configured via `.env` file (auto-generated by setup script):

```env
# Flask
SECRET_KEY=your-secret-key
APP_PORT=8000

# LLM Provider
LLM_PROVIDER=ollama          # or 'openai'
OPENAI_API_KEY=sk-...        # if using OpenAI
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama2

# Twilio (optional)
TWILIO_ACCOUNT_SID=AC...
TWILIO_AUTH_TOKEN=...
TWILIO_PHONE_NUMBER=+1234567890

# Dashboard Theme
DASHBOARD_PRIMARY_COLOR=#0d6efd
DASHBOARD_ACCENT_COLOR=#198754
DASHBOARD_BG_COLOR=#121212
DASHBOARD_CARD_COLOR=#1e1e2e
DASHBOARD_TEXT_COLOR=#e0e0e0
DASHBOARD_TITLE=FraudSim Lab
DASHBOARD_LOGO_TEXT=FSL
```

### LLM Setup

#### Option A: Local Ollama (Recommended for Development)

1. **Install Ollama**: https://ollama.ai
2. **Pull a model**:
   ```bash
   ollama pull llama2
   ```
3. **Start Ollama**:
   ```bash
   ollama serve
   ```
4. **Set in .env**:
   ```env
   LLM_PROVIDER=ollama
   OLLAMA_MODEL=llama2
   ```

#### Option B: OpenAI

1. **Get API key**: https://platform.openai.com/api-keys
2. **Set in .env**:
   ```env
   LLM_PROVIDER=openai
   OPENAI_API_KEY=sk-your-key-here
   ```

### Twilio Setup

1. **Create account**: https://www.twilio.com
2. **Get credentials** from Twilio Console:
   - Account SID
   - Auth Token
   - Phone number (buy or use trial number)
3. **Configure webhooks** in Twilio Console:
   - Voice webhook: `https://your-tunnel-url/voice`
   - Status callback: `https://your-tunnel-url/voice/status`

2. **Customize colors** — Use color picker for theme

## 🔴 Red Flags Detected

The system automatically detects:

| Pattern | Severity | Description |
|---------|----------|-------------|
| Inflated Payroll | HIGH | Unrealistic employee counts or payroll amounts |
| Loan Stacking | HIGH | Multiple loans or applications across banks |
| Fake Address | HIGH | Shell companies or virtual addresses |
| Wire Transfers | HIGH | Requests to send money to accounts |
| Identity Theft | HIGH | Use of false identities or SSNs |
| Urgency | MEDIUM | Pressure to act quickly or before deadlines |
| No Proof | MEDIUM | Missing documentation or records |
| Collusion | MEDIUM | Requests to keep information secret |

## 🐛 Troubleshooting

### Twilio Calls Not Working

- Check `TWILIO_ACCOUNT_SID` and `TWILIO_AUTH_TOKEN` are correct
- Verify phone number is in E.164 format
- Ensure webhook URLs are publicly accessible
- Check Twilio Console for error logs

### LLM Not Responding

**Ollama:**
- Verify Ollama is running: `ollama serve`
- Check `OLLAMA_BASE_URL` is correct
- Ensure model is downloaded: `ollama pull llama2`

**OpenAI:**
- Verify API key is valid
- Check account has available credits
- Review API usage limits

### Database Errors

- Delete `fraudlab.db` to reset database
- Run `python app.py` to reinitialize
- Check file permissions in project directory
